# Nira MVP 产品需求文档

## 产品定位

**Nira** 是面向中国Z世代的活动搭子撮合平台。每周为用户推送 1 个高质量活动搭子匹配，AI 全程代劳安排线下活动（hiking、coffee chat、运动局、看展等）。

产品形态为 **微信小程序**，目标用户为 18-30 岁的一二线城市年轻人。

## 核心理念

- **少即是多**：每周只推 1 个匹配，重质不重量
- **AI 代劳**：从匹配到活动安排，用户零操作负担
- **真实连接**：通过共同活动建立真实社交关系

## 用户流程

```
微信授权登录 → 填写兴趣画像 → 等待周匹配推送
                                    ↓
反馈评价 ← 参加活动 ← AI安排活动细节 ← 收到搭子匹配
    ↓
优化下一轮匹配
```

## MVP 功能范围

### P0 - 必须有

| 功能 | 说明 |
|------|------|
| 微信登录 | 静默授权 + 手机号绑定 |
| 用户画像 | 兴趣标签、活动偏好、时间段、地理位置 |
| 每周匹配 | 基于画像相似度 + 互补性进行 1v1 搭子匹配 |
| AI 活动安排 | 通义千问生成活动方案（时间、地点、活动类型） |
| 匹配通知 | 微信订阅消息推送匹配结果 |
| 活动反馈 | 活动结束后收集双方反馈（评分 + 文字） |

### P1 - 应该有

| 功能 | 说明 |
|------|------|
| 匹配历史 | 查看过往匹配和活动记录 |
| 偏好调整 | 用户主动调整画像和偏好 |
| 活动类型管理 | 后台配置可选活动类型和城市 |

### P2 - 可以有

| 功能 | 说明 |
|------|------|
| 群组活动 | 2v2 或多人活动撮合 |
| 积分/信用体系 | 基于反馈的信任评分 |
| 社区动态 | 活动分享和推荐 |

## 技术架构概述

```
微信小程序 (前端)
    ↓ HTTPS
FastAPI (后端 API)
    ↓
LangGraph (多代理匹配引擎)
    ├── ProfileAgent: 画像分析
    ├── MatchAgent: 匹配计算
    ├── ActivityAgent: 活动方案生成
    └── FeedbackAgent: 反馈处理
    ↓
PostgreSQL + Redis + Chroma
```

## 成功指标（MVP阶段）

- 用户注册 → 完成画像转化率 > 60%
- 收到匹配 → 参加活动转化率 > 40%
- 活动后留存率（次周仍活跃）> 50%
